Serveur MCP Outlook

Connectez Claude Code a votre boite mail et calendrier Microsoft 365. Lisez vos mails, envoyez des reponses, creez des evenements — directement depuis Claude.

Ce que fait ce serveur

Ce serveur ajoute 8 outils a Claude Code pour interagir avec votre compte Microsoft 365 :

OutilDescriptionExemple
authenticateConnexion OAuth2 a votre compte"Connecte-toi a Outlook"
list_emailsLister les emails d'un dossier"Montre-moi mes 10 derniers mails"
read_emailLire un email complet"Lis le mail d'Alexis"
search_emailsRechercher par mots-cles"Cherche les mails de BNP"
send_emailEnvoyer un email"Envoie un mail a Kelly pour..."
create_eventCreer un evenement calendrier"Cree un RDV mercredi a 14h"
list_eventsVoir les prochains evenements"Quels sont mes RDV cette semaine ?"
delete_eventSupprimer un evenement"Annule le RDV de vendredi"
Vos donnees restent privees. Chaque personne se connecte avec son propre compte. Claude n'accede qu'a VOS mails et VOS evenements. Personne d'autre ne peut voir vos donnees.

Etape 1 : Creer l'application Azure

Cette etape est faite une seule fois par un administrateur. Si quelqu'un vous a deja donne un CLIENT_ID, TENANT_ID et CLIENT_SECRET, passez directement a l'etape 2.

1

Ouvrir le portail Azure

Rendez-vous sur https://portal.azure.com et connectez-vous avec votre compte Microsoft 365 professionnel.

2

Aller dans "Inscriptions d'applications"

  1. Dans la barre de recherche en haut, tapez "Inscriptions d'applications" (ou "App registrations" en anglais)
  2. Cliquez sur le resultat "Inscriptions d'applications" sous "Services"
3

Creer une nouvelle application

  1. Cliquez sur le bouton "+ Nouvelle inscription" en haut
  2. Remplissez les champs :
    • Nom : Claude Code Outlook MCP (ou le nom de votre choix)
    • Types de comptes pris en charge : choisissez Comptes dans cet annuaire organisationnel uniquement
    • URI de redirection :
      • Selectionnez "Web" dans le menu deroulant
      • Entrez exactement : http://localhost:3333/callback
  3. Cliquez sur "S'inscrire"
Important : L'URI de redirection doit etre EXACTEMENT http://localhost:3333/callback (avec http, pas https). La moindre difference empechera l'authentification.
4

Recuperer le CLIENT_ID et le TENANT_ID

Apres l'inscription, vous arrivez sur la page "Vue d'ensemble" de l'application. Copiez ces deux valeurs :

  • ID d'application (client) → c'est votre CLIENT_ID
  • ID de l'annuaire (locataire) → c'est votre TENANT_ID
Astuce : Cliquez sur l'icone de copie a droite de chaque valeur pour la copier dans le presse-papiers. Collez-les quelque part temporairement (un bloc-notes par exemple).
5

Creer un secret client

  1. Dans le menu de gauche, cliquez sur "Certificats et secrets"
  2. Cliquez sur "+ Nouveau secret client"
  3. Description : MCP Server
  4. Expiration : 24 mois (recommande)
  5. Cliquez sur "Ajouter"
  6. IMPORTANT : Copiez immediatement la colonne "Valeur" → c'est votre CLIENT_SECRET
Attention : La valeur du secret ne sera visible qu'une seule fois ! Si vous quittez cette page sans la copier, vous devrez en creer un nouveau. Copiez-la MAINTENANT.
6

Configurer les permissions

  1. Dans le menu de gauche, cliquez sur "Autorisations de l'API"
  2. Cliquez sur "+ Ajouter une autorisation"
  3. Choisissez "Microsoft Graph"
  4. Choisissez "Autorisations deleguees"
  5. Dans la barre de recherche, cherchez et cochez :
    • Mail.Read — Lire les emails
    • Mail.Send — Envoyer des emails
    • User.Read — Lire le profil
    • Calendars.ReadWrite — Lire et ecrire le calendrier
  6. Cliquez sur "Ajouter des autorisations"
C'est fait ! Vous avez maintenant vos 3 identifiants : CLIENT_ID, TENANT_ID et CLIENT_SECRET. Gardez-les precieusement pour l'etape suivante.

Etape 2 : Installation

1

Installer Node.js (si pas deja fait)

  1. Allez sur https://nodejs.org
  2. Telechargez la version LTS (bouton vert)
  3. Lancez l'installeur, cliquez "Next" partout, puis "Install"
  4. Verifiez l'installation : ouvrez un terminal (touche Windows, tapez "cmd", Entree) et tapez :
node --version
npm --version

Vous devriez voir des numeros de version (ex: v20.11.0 et 10.2.0). Si ca affiche "n'est pas reconnu", redemarrez votre PC et reessayez.

2

Installer Claude Code (si pas deja fait)

Dans le meme terminal :

npm install -g @anthropic-ai/claude-code
3

Recuperer et copier le dossier du serveur

Ou trouver le dossier ? Votre administrateur ou collegue vous a transmis le dossier outlook-deploy par l'un de ces moyens :

  • Depuis GitHub : telechargez le dossier depuis le repo serumandco/serum-plugin (bouton vert "Code" > "Download ZIP"), puis decompressez-le
  • Depuis un lien partage : OneDrive, Google Drive ou Teams — telechargez et decompressez le .zip
  • Depuis une cle USB ou un dossier reseau : copiez le dossier directement

Ou le placer ? Copiez (ou deplacez) le dossier outlook-deploy ici, et renommez-le en outlook :

C:\Users\VOTRE_NOM\.claude\mcp-servers\outlook\

Remplacez VOTRE_NOM par votre nom d'utilisateur Windows (ex: alexis, safa).

Comment trouver votre nom d'utilisateur ? Ouvrez un terminal (touche Windows, tapez "cmd", Entree) et tapez echo %USERNAME%. C'est le nom qui s'affiche.

Le dossier .claude n'existe pas ? Lancez Claude Code une premiere fois (tapez claude dans le terminal), il creera automatiquement le dossier. Ensuite creez le sous-dossier :

mkdir C:\Users\VOTRE_NOM\.claude\mcp-servers\outlook

Puis copiez tout le contenu du dossier outlook-deploy dedans.

4

Installer les dependances

Dans le terminal, naviguez vers le dossier et installez :

cd C:\Users\VOTRE_NOM\.claude\mcp-servers\outlook
npm install

Attendez que l'installation se termine (peut prendre 30 secondes).

5

Configurer le fichier .env

  1. Dans le dossier outlook, vous trouverez un fichier .env.example
  2. Copiez-le et renommez la copie en .env (sans le "example") 
    copy .env.example .env
  3. Ouvrez le fichier .env avec le Bloc-notes (clic droit > "Ouvrir avec" > Bloc-notes)
  4. Remplacez les valeurs par celles recuperees a l'etape 1 : 
    CLIENT_ID=424e6f78-56e6-4b06-b03b-51e62a429a12
TENANT_ID=ea1c5526-bf6f-4839-88af-a2551ecd29a6
CLIENT_SECRET=IQd8Q~votre-vrai-secret-ici
  5. Sauvegardez (Ctrl+S) et fermez
Ne partagez JAMAIS votre fichier .env ! Il contient vos identifiants confidentiels. Ne l'envoyez pas par email, ne le mettez pas sur GitHub, ne le montrez a personne.
6

Tester le serveur

Dans le terminal :

node index.js

Si tout est bien configure, le programme demarre sans erreur. Fermez-le avec Ctrl+C. Si vous voyez une erreur, verifiez votre fichier .env.

Etape 3 : Configurer Claude Code

Il faut indiquer a Claude Code ou trouver le serveur Outlook.

1

Ouvrir (ou creer) le fichier de configuration

Le fichier se trouve ici :

C:\Users\VOTRE_NOM\.claude\.mcp.json
Le dossier .claude est cache. Pour y acceder : ouvrez l'Explorateur de fichiers, tapez %USERPROFILE%\.claude dans la barre d'adresse, puis Entree.
2

Ajouter la configuration Outlook

Si le fichier n'existe pas, creez-le avec ce contenu :

{
  "mcpServers": {
    "outlook": {
      "command": "node",
      "args": ["C:\\Users\\VOTRE_NOM\\.claude\\mcp-servers\\outlook\\index.js"]
    }
  }
}

Si le fichier existe deja, ajoutez l'entree "outlook" dans la section "mcpServers" :

{
  "mcpServers": {
    "autre-serveur": { ... },
    "outlook": {
      "command": "node",
      "args": ["C:\\Users\\VOTRE_NOM\\.claude\\mcp-servers\\outlook\\index.js"]
    }
  }
}
N'oubliez pas de remplacer VOTRE_NOM par votre nom d'utilisateur Windows. Et attention aux doubles antislash \\ dans le chemin (obligatoire en JSON).
3

Redemarrer Claude Code

Fermez Claude Code completement et relancez-le. Il detectera automatiquement le nouveau serveur.

Etape 4 : Premiere connexion

1

Demander a Claude de se connecter

Dans Claude Code, tapez simplement :

Connecte-toi a Outlook
2

Autoriser dans le navigateur

  1. Votre navigateur s'ouvre automatiquement sur la page de connexion Microsoft
  2. Connectez-vous avec votre compte professionnel
  3. Acceptez les permissions demandees (lecture mails, envoi, calendrier)
  4. Vous verrez une page blanche avec le message "Connecte !"
  5. Fermez l'onglet du navigateur
3

Verifier que ca marche

De retour dans Claude Code, essayez :

Montre-moi mes 5 derniers emails

Claude devrait lister vos emails recents. Si c'est le cas, tout est fonctionnel !

Bravo ! La connexion est sauvegardee. Vous n'aurez plus besoin de vous reconnecter sauf si le token expire (environ tous les 90 jours). Dans ce cas, Claude vous dira "Token expire" et il suffira de refaire Connecte-toi a Outlook.

Etape 5 : Recap quotidien automatique (optionnel)

Vous pouvez configurer un recap automatique de vos emails en attente de reponse, qui apparait dans votre calendrier chaque matin a 9h.

Comment ca marche ?

Un script analyse vos emails recus et envoyes des 7 derniers jours, identifie ceux auxquels vous n'avez pas repondu, et cree un evenement dans votre calendrier avec la liste. Si le recap existe deja pour la journee, il ne cree pas de doublon.

Exemples de demandes a Claude

Lis mes mails de la semaine et dis-moi lesquels attendent une reponse de ma part

Fais-moi un recap de mes emails non lus

Quels sont mes RDV de la semaine ?

Envoie un mail a thomas@kommit.dev pour proposer un call jeudi
Conseil : Demandez a Claude de configurer le script de recap automatique pour vous. Il sait le faire !

Depannage

Si quelque chose ne fonctionne pas, ouvrez la section correspondante ci-dessous :

"Token expire" ou "Non authentifie"

Le token Microsoft a expire (duree de vie ~90 jours). Dans Claude Code, dites simplement Connecte-toi a Outlook pour vous reconnecter. Si ca ne marche pas, supprimez le fichier .msal-cache.json dans le dossier du serveur et reessayez.

"AADSTS700016: Application not found"

Le CLIENT_ID dans votre fichier .env est incorrect. Verifiez-le dans le portail Azure > Inscriptions d'applications > votre app > Vue d'ensemble > "ID d'application (client)".

"AADSTS7000218: request body must contain client_secret"

Le CLIENT_SECRET est manquant ou incorrect dans votre .env. Si vous l'avez perdu, creez un nouveau secret dans Azure > votre app > Certificats et secrets > Nouveau secret client.

"redirect_uri mismatch" ou erreur CORS

L'URI de redirection dans Azure ne correspond pas. Allez dans Azure > votre app > Authentification, et verifiez que l'URI est exactement http://localhost:3333/callback (pas de / final, pas de https).

Le navigateur ne s'ouvre pas automatiquement

Si vous etes en SSH ou dans un environnement sans navigateur, copiez l'URL qui apparait dans les logs de Claude Code et ouvrez-la manuellement dans un navigateur.

"EADDRINUSE" (port 3333 deja utilise)

Un autre programme utilise le port 3333. Fermez-le (souvent une instance precedente du serveur). Sous Windows : ouvrez un terminal et tapez netstat -ano | findstr 3333 pour trouver le processus, puis taskkill /PID [numero] /F pour le fermer.

"Variables manquantes dans le fichier .env"

Vous n'avez pas cree le fichier .env ou il est mal rempli. Verifiez qu'il existe dans le meme dossier que index.js et qu'il contient bien CLIENT_ID, TENANT_ID et CLIENT_SECRET sans espaces autour du =.

"node n'est pas reconnu comme commande"

Node.js n'est pas installe ou pas dans le PATH. Reinstallez Node.js depuis https://nodejs.org (version LTS), redemarrez votre PC, puis reessayez.

Claude ne detecte pas le serveur Outlook

Verifiez que :
1. Le fichier .mcp.json est bien dans C:\Users\VOTRE_NOM\.claude\
2. Le chemin vers index.js est correct (avec des doubles antislash \\)
3. Vous avez redemarre Claude Code apres la modification
4. npm install a ete execute dans le dossier du serveur

Pour les administrateurs

Si vous etes administrateur Microsoft 365 de votre organisation, vous pouvez autoriser l'application pour tous vos collegues en un clic.

Accorder le consentement pour toute l'organisation

  1. Allez dans le portail Azure > Inscriptions d'applications > votre app
  2. Cliquez sur "Autorisations de l'API" dans le menu de gauche
  3. Cliquez sur le bouton "Accorder le consentement de l'administrateur pour [votre organisation]"
  4. Confirmez
Effet : Vos collegues n'auront plus la popup "Accepter les permissions" lors de leur premiere connexion. L'authentification sera transparente.

Distribuer aux collegues

Partagez avec chaque collegue :

  1. Le dossier outlook-deploy (sans le .env !)
  2. Les 3 identifiants : CLIENT_ID, TENANT_ID, CLIENT_SECRET (par message prive)
  3. Ce guide (le fichier README.html)
Securite : Le CLIENT_SECRET donne acces aux permissions de l'app. Transmettez-le uniquement aux personnes de confiance, de preference via un canal securise (pas par email non chiffre).

Rotation du secret

Le secret client expire apres la duree choisie (12 ou 24 mois). Pensez a le renouveler avant expiration :

  1. Azure > votre app > Certificats et secrets > Nouveau secret client
  2. Communiquez la nouvelle valeur a tous les utilisateurs
  3. Chacun met a jour son fichier .env
  4. Supprimez l'ancien secret dans Azure
